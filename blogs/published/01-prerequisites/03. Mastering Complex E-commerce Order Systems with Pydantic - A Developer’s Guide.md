## The Foundation: Money Handling Done Right

Financial data is the backbone of any e-commerce system, and getting it wrong isn’t an option. Here’s how to create a bulletproof money model:

from pydantic import BaseModel, confloat, constr  
  
class Money(BaseModel):  
    amount: confloat(ge=0)  # Ensures non-negative amounts  
    currency: constr(min_length=3, max_length=3)  # ISO currency codes  
      
    def __str__(self):  
        return f"{self.amount} {self.currency}"  

This simple model prevents common financial disasters like negative amounts and invalid currency codes. The `confloat(ge=0)` constraint ensures you'll never accidentally process a -$500 refund as a charge.

## Order Processing: Where Complexity Meets Elegance

The real magic happens in the order models. Here’s how to handle order items with automatic total calculations:

from pydantic import conint  
from uuid import UUID, uuid4  
  
class OrderItem(BaseModel):  
    quantity: conint(gt=0)  # Must be positive  
    unit_price: Money  
      
    @property  
    def total_price(self) -> Money:  
        return Money(  
            amount=self.quantity * self.unit_price.amount,  
            currency=self.unit_price.currency  
        )

The `@property` decorator creates computed fields that automatically calculate totals while maintaining immutability. No more manual calculation errors or currency mismatches.

## The Complete Order Model: Enterprise-Ready

class OrderStatus(str, Enum):  
    PENDING = "pending"  
    CONFIRMED = "confirmed"   
    SHIPPED = "shipped"  
    DELIVERED = "delivered"  
    CANCELLED = "cancelled"  
  
class Order(BaseModel):  
    id: UUID = Field(default_factory=uuid4)  
    order_number: str  
    customer_id: UUID  
    status: OrderStatus = OrderStatus.PENDING  
    items: List[OrderItem]  
    shipping_address: ShippingAddress  
    billing_address: ShippingAddress  
    subtotal: Money  
    tax_amount: Money  
    shipping_cost: Money  
    discount_amount: Money = Field(default=Money(amount=0, currency="USD"))  
    total: Money  
    notes: Optional[str] = None  
    created_at: datetime = Field(default_factory=datetime.now)  
    updated_at: Optional[datetime] = None

This model captures every aspect of a complex order while providing sensible defaults and automatic timestamp generation.

## Advanced Validation Patterns

Add custom business logic validation with field validators:

@field_validator('order_number')  
@classmethod  
def validate_order_number(cls, v):  
    if not v.startswith('ORD-'):  
        raise ValueError('Order Number must start with "ORD-"')  
    return v  
  
@model_validator(mode='after')  
def validate_financial_consistency(self):  
    calculated_total = (  
        self.subtotal.amount +   
        self.tax_amount.amount +   
        self.shipping_cost.amount -   
        self.discount_amount.amount  
    )  
    if abs(calculated_total - self.total.amount) > 0.01:  
        raise ValueError('Financial totals do not match')  
    return self
